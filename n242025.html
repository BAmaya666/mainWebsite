Devfest Vancouver 2025

November 24, 2025

Last Thursday, on the 20th, I attended the Google Devfest 2025, organized by the Google Developer Group in Vancouver. The main organizers were Procheta Nag and Dinuka Gunaratne.

First of all, here are some of the key concepts I learned âœ¨

  • Framework / Library: A pre-fabricated kit that provides a structured foundation for building software applications. It typically includes:
    • Pre-written code for common tasks (like handling user input, managing files, or communicating with a database).
    • A set of rules and conventions for code structure.
    • o Tools and libraries that make development faster and more consistent.
  • React: A JavaScript library (often referred to as a framework) for building user interfaces, primarily for websites.
  • React Native: A JavaScript framework for building natively rendering mobile applications for iOS and Android.
    • Native App: An application compiled into the native language of its platform (e.g., Objective-C/Swift for iOS, Java/Kotlin for Android). This provides direct access to device features (camera, GPS, contacts) and delivers the performance and user experience of an app built specifically for that operating system.
  • Expo:A set of tools and services built around React Native to streamline development, providing a smoother experience, especially for beginners and for building prototypes.
  • Cursor: An AI-powered code editor designed to help you write code faster using AI assistance.
  • Firebase: A Backend-as-a-Service (BaaS) platform owned by Google that provides ready-to-use cloud services, eliminating the need to build your own server infrastructure from scratch.
  • N8n: A workflow automation platform that combines AI capabilities to help you connect different apps and services, automating tasks without writing code. It is part of the "integration platform as a service" (iPaaS) ecosystem.
  • Zapier: A workflow automation platform similar to n8n. It enables users to connect web applications, automate tasks, and incorporate AI into their workflows.
  • Gemini CLI: An AI agent that provides direct access to Gemini from your terminal.
  • Gemini Canvas: A workspace within the Gemini app where users can create, edit, and refine documents and code with AI assistance. It functions as a real-time, unified environment for writing, coding, brainstorming, and prototyping.
  • Claude: An AI assistant built by Anthropic.
  • MCP (Model Context Protocol): An open-source standard for connecting AI applications to external systems. It allows AIs like Claude or ChatGPT to connect to data sources (e.g., local files, databases), tools (e.g., search engines), and workflows, enabling them to access information and perform tasks.

Talks and Workshop

The first talk focused on Gemini and its new features, such as Gemini CLI and Gemini Canvas. It was presented by Richard Seroter, a Senior Director at Google Cloud. He mentioned that Gemini Canvas is similar to ChatGPT apps, where you can use apps displayed inside the ChatGPT conversation and use them as an appendix of ChatGPT.

He also discussed Google Antigravity, an ADK (Agent Development Kit) featuring an IDE (similar to the shell Editor in Google Cloud or VSCode). Its advantage is a CLI that understands natural language, functioning as if you are a project manager delegating tasks to autonomous software "agents."

Next, Taylor Stacey, a Senior AI Architect, explained how to build and deploy an ADK agent that uses an MCP inside Google Cloudâ€”a process we later replicated in the workshop called Agentverse. He emphasized the importance of writing documentation throughout projects to improve collaboration and to reduce errors.

Afterwards, Morten Rand-Hendriksen demonstrated how to connect any project with MCPs. He showed us how to connect Claude to Zapier and manipulate project data from the server using the MCP tool.

In the second half, after lunch, we participated in the Agentverse workshop. Attendees were divided into four teams (Web Developers, DevOps Engineers, Data Engineers, and Architects) to build a project on Google Cloud. We used the terminal and Gemini to accelerate the development process.

We coded in Python, a language I am not familiar with, but the workshop was structured so we could simply copy, paste, and follow instructions to see the project come to life. It was impressive to see how we could code, test everything in the cloud, and use Gemini to prompt the system.

I must note that at one point, my code started looping, and Gemini began automatically updating, erasing, and rewriting values, likely due to a bug or a step I had skipped. However, since many others finished their projects successfully, it was probably the latter.

Between the coffee breaks and the networking lunch, I had the chance to talk with a diverse group of attendees. There were students seeking internships, senior engineers with years of experience, people from different careers who code for fun, and professionals who use AI to write code for their jobs.

One person told me he uses Firebase and was able to deploy nine apps in about a month. Similarly, some friends mentioned Expo and React Native as comparable frameworks.

A very young woman mentioned she codes in any language using Cursor and that her team recently won first place in a hackathon.

Another attendee, a greenhouse manager, explained that he simply asks Claude or ChatGPT how to get a daily weather forecast message on his phone or how to create an AR program to spot crooked stems on his pepper crops.

That day was truly amazing. Speaking with all those skilled individuals, for whom every conversation was a source of valuable knowledge, and realizing that my friends are brilliant in their fields, encouraged me to continue on this programming path. It was also overwhelming to encounter so much information that seems to be common knowledge, yet I was unaware of it. However, I am positive that tech knowledge is exponentialâ€” it's a rabbit hole where grasping one topic naturally leads you to the next.

My definitive takeaway is to learn AI, master it, and test and play with more tools. I will keep seeking out these kinds of events and communities, and volunteer if necessary. Earlier this year, I was experimenting with uncensored, portable LLMs, and for a long time, I've wanted to automate my house using an "ocarina" like in Zelda. This experience is my call to action to finally get my hands busy with these projects.
